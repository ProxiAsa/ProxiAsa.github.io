Ferme RDS Mairie
1 Broker — 2 Session Hosts — FSLogix

Déploiement d'une ferme Remote Desktop Services pour une mairie cliente d'Aintranet. Architecture simple et efficace avec un broker, deux Session Hosts et FSLogix pour la gestion des profils utilisateurs — permettant aux agents de retrouver leur environnement identique quel que soit le serveur sur lequel ils atterrissent.

Réalisé
📅 2025
🏢 Aintranet — Client mairie
RDS Windows Server 2022 FSLogix Active Directory GPO RemoteApp Partage SMB

Contexte & objectifs

Une mairie cliente d'Aintranet souhaitait centraliser l'accès à ses applications métier (logiciel de gestion communale, suite bureautique) pour une trentaine d'agents. Le besoin principal : que chaque agent retrouve son environnement personnel (bureau, favoris, paramètres applicatifs) quel que soit le Session Host sur lequel il se connecte — c'est précisément ce que FSLogix résout.

Client

Mairie — ~30 agents

Profils

FSLogix Profile Containers stockés sur partage SMB

Architecture

1 Broker + 2 Session Hosts + 1 partage de profils NAS

Environnement

Windows Server 2022, AD existant de la mairie

Qu'est-ce que FSLogix ?

📁 FSLogix Profile Containers

FSLogix est une solution Microsoft de gestion de profils utilisateurs pour les environnements RDS et VDI. Au lieu de copier le profil à l'ouverture de session (comme les profils itinérants classiques), FSLogix monte un disque virtuel VHD(X) contenant l'intégralité du profil de l'utilisateur directement depuis un partage réseau.

Avantages clés : ouverture de session rapide (<10s), profil cohérent sur tous les Session Hosts, pas de problème de synchronisation, support natif Office 365 et Outlook (cache OST inclus dans le container).

Stockage : Chaque utilisateur possède un fichier NomPrenom_SID.vhdx sur le partage SMB, monté automatiquement à la connexion et démonté à la déconnexion.

Profils itinérants classiques
  • Copie du profil à chaque connexion
  • Lente sur gros profils
  • Conflits de synchro fréquents
  • OST Outlook non supporté
  • Problèmes avec Office 365
FSLogix Profile Containers
  • Montage VHD(X) à la connexion
  • Rapide quelle que soit la taille
  • Aucun conflit de synchro
  • OST Outlook inclus dans le container
  • Compatible Office 365 nativement

Architecture déployée

Agents mairie / Postes clients | RDP / RemoteApp [RD Gateway] — Accès sécurisé, certificat SSL | [RD Web Access] — Portail applications mairie | [RD Connection Broker] — Répartition de charge ____|____ | | [RDSH-01] [RDSH-02] — Session Hosts Windows Server 2022 | | |_________| | SMB (montage VHD(X) profil) [Partage FSLogix \\SRV\Profils\] — NAS ou serveur de fichiers | [Active Directory / GPO] — Auth, groupes FSLogix, politiques

Étapes de déploiement

  • 01
    Préparation de l'AD existant — Création des OUs pour les serveurs RDS et les utilisateurs. Création du groupe de sécurité GRP_FSLogix_Users. Attribution des licences CAL RDS.
  • 02
    Installation des rôles RDS — Via Server Manager, déploiement des rôles RD Connection Broker, RD Session Host (x2), RD Gateway et RD Web Access sur les serveurs désignés sous Windows Server 2022.
  • 03
    Création de la collection de sessions — Configuration d'une collection unique avec les deux Session Hosts. Load balancing équilibré (50/50). Publication des applications métier en RemoteApp.
  • 04
    Préparation du partage FSLogix — Création du partage SMB \\SRV-FICHIERS\Profils$. Permissions NTFS : les comptes machines des Session Hosts et le groupe GRP_FSLogix_Users ont le contrôle total. Les utilisateurs n'ont pas d'accès direct au dossier racine.
  • 05
    Installation de FSLogix sur les Session Hosts — Téléchargement de l'agent FSLogix depuis le portail Microsoft (inclus dans les licences Microsoft 365). Installation sur RDSH-01 et RDSH-02.
  • 06
    Configuration FSLogix via GPO — Création d'une GPO liée à l'OU des Session Hosts. Paramètres clés : Enabled = 1, VHDLocations = \\SRV-FICHIERS\Profils$, DeleteLocalProfileWhenVHDShouldApply = 1. Type VHD(X) sélectionné pour de meilleures performances.
  • 07
    Configuration RD Gateway & certificat SSL — Génération et installation du certificat SSL. Politiques CAP (qui peut se connecter) et RAP (vers quelles ressources) configurées par groupe AD.
  • 08
    GPO complémentaires — Redirection du dossier Documents vers le serveur de fichiers (en complément du profil FSLogix), timeout de session après 30 min d'inactivité, restrictions logicielles sur les Session Hosts.
  • 09
    Tests & validation — Connexion d'un agent sur RDSH-01 : vérification de la création du NomPrenom_SID.vhdx sur le partage. Déconnexion et reconnexion sur RDSH-02 : le profil est identique. Test de montée en charge avec 10 sessions simultanées.

Configuration FSLogix — paramètres clés

🌏 Clé de registre / GPO (HKLM\SOFTWARE\FSLogix\Profiles)

Enabled = 1 — Active FSLogix Profile Containers

VHDLocations = \\SRV-FICHIERS\Profils$ — Chemin du partage SMB

VolumeType = VHDX — Format de disque virtuel (VHDX recommandé)

SizeInMBs = 30000 — Taille max du container (30 Go)

DeleteLocalProfileWhenVHDShouldApply = 1 — Supprime l'éventuel profil local résiduel

FlipFlopProfileDirectoryName = 1 — Format de dossier Prénom_NOM_SID plus lisible

🔒 Permissions NTFS sur le partage : Les comptes machines RDSH-01$ et RDSH-02$ doivent avoir le contrôle total sur le partage racine pour pouvoir créer et monter les VHD(X) des utilisateurs.
